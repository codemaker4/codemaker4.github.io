Update: JSowlrdGen should now work faster

The terrain generation of JSworldgen used square roots to calculate the distance between each pixel an each heightpoint. Square roots need much time to be calculated, and because it needs to calculate +- 3000 distances per pixel, it slows down the algorithim quite a bit. You can also calculate the distance by simply not putting the sqrt() around the scuared distances along the 2 axises. This does result in the new function not returning the actual distance, but a much heigher value that is still usable. Also the max amount of calculations per frame is now not limited by a fixed value, but instead the algorithim keeps on calculating new pixels untill 25ms since the last frame have passed. This results in a max framerate of 40FPS, wich is enough to my opinion.

Update: Math plotter can now show RGB colors

I updated MathPoltter to now accept arrays of 3 items and show the values of the items in different colors. This only applies when using something like "return [Redval, Blueval, Greenval]". You can now only give integers, but in the future you can also return booleans in the array. When returning only a integer or a boolean, the old behavior applies. All old functions should still work.
You can with some smart JS still use booleans with the new color channels. This can be done by first calculation the boolean and next use an if-statement to turn it into an integer. This function plots a red and a green line, with a blue circle at (0,0). "var myBlue = (dist(0,0,x,y) < 100); if (myBlue) {myBlue = 255;} else {myBlue = 0;} return [abs(y-(x*2+5))*-1+255, abs(y-(x*-2-5))*-1+255, myBlue]"
I also fixed a bug: Math returning integers was plotted upsideDown. The solution also improves the performance ;).

releasing math plotter

I modified JSWorldGen to now plot any mathematical equation. You can move around by swiping/dragging or using WASD. To set the shown formula, press the + icon on the top left and enter a javascript function in the popup. For example "return x+y" or "return y/(x%255)". The p5.js library is also available in these functions, for example "return dist(0,0,x,y)" or "return noise(x/100, y/100)*255". NOTE: if your function returns a integer, it must be between 0 and 255. if not, the value will be lowerd/higherd to be inside this range. TIP: at the end of your function add "%255". this makes sure that the returned value is in between 0 and 255. Try "return dist(0,0,x,y)%255" to understand it's effect better.
You can also let your function return booleans, for example "return x == y" or "return round(dist(0,0,x,y)) == 100"
